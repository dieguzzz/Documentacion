Documentación

Introducción

Este documento proporciona una descripción general del proyecto de gestión basado en la web para una institución educativa ITSE.

Se destacarán las funcionalidades clave, la estructura del proyecto y la interacción con la base de datos.

Sección 1: Estructura del Proyecto

El directorio principal es parcial1, que contiene todos los archivos fuente y recursos.

La carpeta .idea/ incluye configuraciones específicas para el IDE.

Dentro de controladores/, se encuentran scripts PHP para manejar operaciones de backend, como agregar y eliminar registros.

Archivos clave incluyen INDEX.php (punto de entrada), INICIAR_SESION.php (inicio de sesión) y REGISTRO.php (registro de usuarios).

Otros archivos y directorios importantes incluyen:

  • cerrar_sesion.php: Maneja el cierre de sesión de los usuarios.
  • dejar_tarea.php: Permite a los profesores dejar tareas para los estudiantes.
  • MAPA.php: Muestra un mapa interactivo de las instalaciones de la institución.
  • SISTEMA_GESTION_HT.php, SISTEMA_GESTION_ID.PHP, SISTEMA_GESTION_NG.php, SISTEMA_GESTION_TI.php: Páginas para la gestión de diferentes departamentos o sistemas dentro de la institución.
  • CSS1.css: Archivo principal de estilos CSS.
  • Videos/, FONTS/, Fotos/: Directorios que contienen videos, fuentes e imágenes utilizados en la aplicación.

Sección 2: Funcionalidades Clave

Autenticación de Usuarios

Gestión de inicio de sesión y registro, restringiendo acceso a ciertas páginas.

  • INICIAR_SESION.php: Página de inicio de sesión que verifica las credenciales del usuario y establece las variables de sesión.
  • REGISTRO.php: Página de registro que permite a los nuevos usuarios crear una cuenta.
  • cerrar_sesion.php: Script que maneja el cierre de sesión de los usuarios.

Gestión de Datos

Scripts para interactuar con diferentes tablas en la base de datos.

  • anadir-HT.php, anadir-ID.php, anadir-NG.php, anadir-TI.php: Scripts para agregar nuevos registros a las tablas correspondientes.
  • eliminar_HT.php, eliminar_ID.php, eliminar_NG.php, eliminar_TI.php: Scripts para eliminar registros de las tablas correspondientes.
  • controlador_HT.php: Script para obtener y mostrar datos de la tabla de profesores.
  • controlar-editar_HT.php, controlar-editar_ID.php, controlar-editar_NG.php, controlar-editar_TI.php: Scripts para actualizar registros en las tablas correspondientes.

Navegación y UI

Uso de Bootstrap para diseño responsivo y una experiencia de usuario mejorada.

  • INDEX.php: Página principal que incluye una barra de navegación, un video de fondo y un botón para iniciar un recorrido.
  • MAPA.php: Página que muestra un mapa interactivo de las instalaciones de la institución.
  • dejar_tarea.php: Página que permite a los profesores dejar tareas para los estudiantes.
  • tareas.php: Página que muestra una lista de tareas disponibles para los usuarios.

Explicación del Flujo de Datos

1. Inicio

Usuario accede a la página principal INDEX.php.

Desde aquí, el usuario puede:

  • Iniciar sesión (INICIAR_SESION.php).
  • Registrarse (REGISTRO.php).
  • Ver el mapa de escuelas (MAPA.php).

2. Iniciar Sesión

El usuario ingresa sus credenciales en INICIAR_SESION.php.

El sistema verifica las credenciales:

  • Si son correctas, redirige al usuario a tareas.php.
  • Si son incorrectas, muestra un mensaje de error.

3. Registrarse

El usuario completa el formulario de registro en REGISTRO.php.

El sistema procesa los datos de registro:

  • Si el registro es exitoso, redirige al usuario a INICIAR_SESION.php.
  • Si hay un error, muestra un mensaje de error.

4. Mapa de Escuelas

El usuario puede seleccionar una escuela en MAPA.php.

Dependiendo de la selección, el usuario es redirigido a:

  • SISTEMA_GESTION_ID.php para Innovación Digital.
  • SISTEMA_GESTION_NG.php para Negocios.
  • SISTEMA_GESTION_HT.php para Hospitalidad y Turismo.
  • SISTEMA_GESTION_TI.php para Tecnología Industrial.

5. Gestión de Tareas

En tareas.php, el usuario puede:

  • Ver las tareas asignadas.
  • Dejar una nueva tarea (dejar_tarea.php).
  • Cerrar sesión (cerrar_sesion.php).

6. Dejar Tarea

El usuario completa el formulario para dejar una tarea en dejar_tarea.php.

El sistema procesa los datos de la tarea:

  • Si la tarea se registra exitosamente, redirige al usuario a tareas.php.
  • Si hay un error, muestra un mensaje de error.

7. Cerrar Sesión

El usuario cierra sesión en cerrar_sesion.php.

El sistema destruye la sesión y redirige al usuario a INICIAR_SESION.php.

Conclusión

Este proyecto representa un sistema de gestión integral que facilita la autenticación de usuarios y la gestión de datos, cumpliendo con los objetivos de proporcionar una plataforma eficiente para la institución educativa.

La estructura del proyecto está bien organizada, con una clara separación entre los scripts de backend y los archivos de frontend.

El uso de Bootstrap y otros recursos externos mejora la experiencia del usuario y asegura que la aplicación sea responsiva y fácil de usar.
